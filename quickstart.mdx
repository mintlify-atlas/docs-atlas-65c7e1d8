This guide walks you through cloning G5API, configuring it, running database migrations, and verifying the server is up — the fastest path to a working instance.

## Prerequisites

Before you begin, make sure the following are available on your machine:

- **Node.js** `>=16.17.0` — required by the `engines` field in `package.json`
- **MariaDB or MySQL** — G5API uses `db-migrate-mysql` and requires a compatible database server
- **Yarn** — used for all package and script management
- **Redis** *(optional)* — enables session caching; set `useRedis: false` in config to skip it
- **A Steam API key** — required for Steam OAuth authentication; obtain one at the Steam developer portal

## Get running

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/PhlexPlexico/G5API.git
    cd G5API
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    yarn
    ```
  </Step>

  <Step title="Copy the config template">
    Copy the development config template and open it for editing:

    ```bash
    cp config/development.json.template config/development.json
    ```

    Then fill in the required values. The template ships with sensible defaults for local development:

    ```json config/development.json
    {
      "server": {
        "port": 3301,
        "hostname": "http://localhost",
        "dbKey": "Database 16 Byte Key.",
        "steamAPIKey": "API Key For Steam Calls.",
        "sharedSecret": "a secure secret for session sigining.",
        "clientHome": "http://localhost:8080",
        "useRedis": true,
        "apiURL": "http://localhost:8080/api/",
        "uploadDemos": false,
        "localLoginEnabled": true,
        "redisUrl": "redis://:super_secure@localhost:6379",
        "redisTTL": 86400
      },
      "development": {
        "driver": "mysql",
        "user": "get5_user",
        "password": "",
        "database": "get5dev",
        "multipleStatements": true,
        "flags": { "ENV": "MYSQL_FLAGS" },
        "host": "127.0.0.1",
        "port": 3306,
        "connectionLimit": 15
      },
      "admins": {
        "steam_ids": "admins,go,here"
      },
      "super_admins": {
        "steam_ids": "super_admins,go,here"
      }
    }
    ```

    At minimum, replace these values:

    | Key | Description |
    |-----|-------------|
    | `steamAPIKey` | Your Steam Web API key |
    | `dbKey` | A 16-byte AES encryption key for the database |
    | `sharedSecret` | A secret string used to sign sessions |
    | `development.user` / `development.password` | Credentials for your MariaDB/MySQL user |
    | `super_admins.steam_ids` | Comma-separated Steam IDs that get super admin access |

    <Note>
      If you are not running Redis locally, set `"useRedis": false`. The server will fall back to in-memory sessions.
    </Note>
  </Step>

  <Step title="Run database migrations">
    Create the development database and apply all migrations:

    ```bash
    yarn migrate-create-dev && yarn migrate-dev-upgrade
    ```

    <Warning>
      `migrate-create-dev` drops and recreates the `get5dev` database. Do not run this against a database that contains data you want to keep.
    </Warning>
  </Step>

  <Step title="Start the development server">
    ```bash
    yarn start
    ```

    This compiles TypeScript with `tsc-watch` and starts the server via `nodemon`. The server listens on port `3301` by default.

    <Tip>
      The `NODE_ENV` variable controls which config file is loaded. The `yarn start` script defaults to `development`, so `config/development.json` is used.
    </Tip>
  </Step>

  <Step title="Verify the server is running">
    Open a second terminal and run:

    ```bash
    curl http://localhost:3301/isloggedin
    ```

    You should receive a JSON response indicating the login status. A `401` or similar structured JSON response (rather than a connection error) confirms the server is up and handling requests.
  </Step>
</Steps>

## Explore the API

G5API ships with a Swagger UI that documents every endpoint. Once the server is running, open your browser and navigate to:

```
http://localhost:3301/api-docs
```

This interactive UI lets you browse all available routes, inspect request/response schemas, and make test calls directly from the browser.
