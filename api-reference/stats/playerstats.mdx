---
title: "Player Stats"
description: "Per-player and per-map statistics for CS2 matches, including detailed kill-event data."
---

G5API tracks two layers of player statistics:

- **Player Stats** (`/playerstats`) — aggregated per-map counters: kills, deaths, assists, multi-kills, clutches, damage, etc.
- **Extra Stats** (`/playerstatsextra`) — per-death event records: weapon, headshot, smoke, blind, assister, etc.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/playerstats` | All player stats in the system |
| GET | `/playerstats/unique` | Count of distinct players who have participated |
| GET | `/playerstats/match/:match_id` | All player stats for a match |
| GET | `/playerstats/match/:match_id/stream` | Real-time SSE stream for a match |
| GET | `/playerstats/:steam_id` | All stats rows for a Steam64 ID |
| GET | `/playerstats/:steam_id/pug` | Aggregated PUG stats for a player |
| GET | `/playerstats/:steam_id/official` | Aggregated official-match stats for a player |
| GET | `/playerstats/:steam_id/season/:season_id` | Season-scoped aggregated stats |
| GET | `/playerstats/:steam_id/recent` | Last 5 matches the player participated in |
| GET | `/playerstatsextra` | All extra (per-death) stats |
| GET | `/playerstatsextra/:steam_id` | Extra stats for a Steam64 ID |
| GET | `/playerstatsextra/:steam_id/pug` | Extra PUG stats for a player |
| GET | `/playerstatsextra/:steam_id/official` | Extra official-match stats for a player |
| GET | `/playerstatsextra/:steam_id/season/:season_id` | Season-scoped extra stats |
| GET | `/playerstatsextra/match/:match_id` | Extra stats for a match |
| GET | `/playerstatsextra/match/:match_id/stream` | Real-time SSE stream of extra stats |

## Real-time Stream — GET /playerstats/match/:match_id/stream

Streams live player stat updates for a match as an SSE event stream.

```bash
curl -N -H "Accept: text/event-stream" \
  https://your-g5api.example.com/playerstats/match/42/stream
```

Events use event name `playerstats` and carry a JSON array of all player stat rows for the match.

```
event: playerstats
data: [{"match_id":42,"steam_id":"765...","kills":5,...}]
```

## Extra Stats — /playerstatsextra

Extra stats store one record per player death event. This enables detailed analysis like spray patterns, economy tracking, and multi-kill reconstruction.

### GET /playerstatsextra

```bash
curl https://your-g5api.example.com/playerstatsextra
```

**Response `200`**

```json
{
  "playerStatExtra": [
    {
      "id": 1,
      "match_id": 42,
      "map_id": 1,
      "team_id": 3,
      "player_steam_id": "76561198012345678",
      "player_name": "playerOne",
      "player_side": "ct",
      "round_number": 5,
      "round_time": 42000,
      "attacker_steam_id": "76561198087654321",
      "attacker_name": "playerTwo",
      "attacker_side": "t",
      "weapon": "weapon_ak47",
      "bomb": false,
      "headshot": true,
      "thru_smoke": false,
      "attacker_blind": false,
      "no_scope": false,
      "suicide": false,
      "friendly_fire": false,
      "assister_steam_id": null,
      "assister_name": null,
      "assister_side": null,
      "assist_friendly_fire": false,
      "flash_assist": false
    }
  ]
}
```

<ResponseField name="playerStatExtra" type="array">
  <Expandable title="PlayerStatsExtra fields">
    <ResponseField name="player_steam_id" type="string">Steam64 ID of the player who died.</ResponseField>
    <ResponseField name="player_name" type="string">In-game name of the player who died.</ResponseField>
    <ResponseField name="player_side" type="string">Side the victim was on (`ct` or `t`).</ResponseField>
    <ResponseField name="match_id" type="integer" required>Parent match ID.</ResponseField>
    <ResponseField name="map_id" type="integer" required>Parent map stat ID.</ResponseField>
    <ResponseField name="team_id" type="integer" required>Team ID of the victim.</ResponseField>
    <ResponseField name="round_number" type="integer">Round the death occurred in.</ResponseField>
    <ResponseField name="round_time" type="integer">Milliseconds into the round when death occurred.</ResponseField>
    <ResponseField name="attacker_steam_id" type="string">Steam64 ID of the attacker.</ResponseField>
    <ResponseField name="attacker_name" type="string">In-game name of the attacker.</ResponseField>
    <ResponseField name="attacker_side" type="string">Side the attacker was on.</ResponseField>
    <ResponseField name="weapon" type="string">Weapon classname (e.g. `weapon_ak47`).</ResponseField>
    <ResponseField name="bomb" type="boolean">Whether death was caused by the bomb explosion.</ResponseField>
    <ResponseField name="headshot" type="boolean">Whether the killing shot was a headshot.</ResponseField>
    <ResponseField name="thru_smoke" type="boolean">Whether the kill was through smoke.</ResponseField>
    <ResponseField name="attacker_blind" type="boolean">Whether the attacker was blinded when they fired.</ResponseField>
    <ResponseField name="no_scope" type="boolean">Whether the kill was a no-scope.</ResponseField>
    <ResponseField name="suicide" type="boolean">Whether the player killed themselves.</ResponseField>
    <ResponseField name="friendly_fire" type="boolean">Whether the kill was from a teammate.</ResponseField>
    <ResponseField name="assister_steam_id" type="string">Steam64 ID of the assist player. `null` if no assist.</ResponseField>
    <ResponseField name="assister_name" type="string">Name of the assister.</ResponseField>
    <ResponseField name="assister_side" type="string">Side of the assister.</ResponseField>
    <ResponseField name="assist_friendly_fire" type="boolean">Whether the assist was friendly fire.</ResponseField>
    <ResponseField name="flash_assist" type="boolean">Whether the assist was via flashbang.</ResponseField>
  </Expandable>
</ResponseField>
