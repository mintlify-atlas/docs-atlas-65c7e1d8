G5API uses the [node-config](https://github.com/node-config/node-config) package to manage configuration. At startup, node-config reads a JSON file from the `config/` directory whose name matches the `NODE_ENV` environment variable. You never edit `app.ts` to change settings — all tunables live in these JSON files.

## Config files

Three environment files exist in the `config/` directory. Each corresponds to a `NODE_ENV` value:

| File | `NODE_ENV` value | Typical use |
|------|-----------------|-------------|
| `development.json` | `development` | Local development with hot-reload via `tsc-watch` |
| `production.json` | `production` | Docker / PM2 deployments |
| `test.json` | `test` | Automated test suite (`yarn test`) |

<Note>
The repository only ships `.template` files. You must copy the template to create the actual file before starting the server.

```bash
cp config/development.json.template config/development.json
```
</Note>

## Top-level config sections

Every config file has four top-level sections:

| Section | Purpose |
|---------|---------|
| `server` | HTTP server, session, Steam, Redis, and CORS settings |
| `development` / `production` / `test` | Database connection (the key matches `NODE_ENV`) |
| `admins` | Comma-separated Steam IDs granted admin privileges |
| `super_admins` | Comma-separated Steam IDs granted super-admin privileges |

## Environment variable overrides via Docker

The production template uses shell variable placeholders (`$PORT`, `$STEAMAPIKEY`, etc.). The Docker image runs `envsubst` at container startup to substitute these placeholders with real values from container environment variables:

```dockerfile
CMD envsubst < /Get5API/config/production.json.template > /Get5API/config/production.json && \
    yarn migrate-create-prod && \
    yarn migrate-prod-upgrade && \
    yarn startprod
```

Set each variable in your `docker-compose.yml` under the `environment` key:

```yaml
environment:
  - NODE_ENV=production
  - PORT=3301
  - HOSTNAME=https://yourdomain.com
  - DBKEY=your_16_byte_db_key
  - STEAMAPIKEY=your_steam_api_key
  - SHAREDSECRET=your_session_secret
  - CLIENTHOME=https://yourdomain.com
  - APIURL=https://yourdomain.com/api
  - SQLUSER=get5
  - SQLPASSWORD=your_db_password
  - SQLPORT=3306
  - DATABASE=get5
  - SQLHOST=get5db
  - ADMINS=76561198000000001,76561198000000002
  - SUPERADMINS=76561198000000003
  - REDISURL=redis://:your_redis_password@redis:6379
  - REDISTTL=86400
  - USEREDIS=true
  - UPLOADDEMOS=true
  - LOCALLOGINS=false
```

<Tip>
For local development you do not need Docker. Copy `development.json.template` to `development.json`, fill in the values directly, and run `yarn start`.
</Tip>
